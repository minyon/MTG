# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Import all decks from import/ folder (.txt and .dck files)
python3 import_deck.py

# Import a single deck file
python3 import_deck.py <file.txt>

# Import with an explicit deck name
python3 import_deck.py <file.txt> <deck-name>

# Analyze a deck against Scryfall (by slug or path)
python3 analyze_deck.py <deck-slug>
python3 analyze_deck.py <path/to/deck.txt>
```

No build step, test suite, or package dependencies — pure Python stdlib. `analyze_deck.py` requires internet access (calls Scryfall API).

## Architecture

Two scripts, one deck storage convention:

**`import_deck.py`** — Converts plain-text deck exports into the canonical per-deck directory structure. Supports Moxfield and Deckstats input formats (auto-detected). Strips Moxfield timestamps from filenames to derive the slug (e.g. `norin-bounce-20250519-061823.txt` → `norin-bounce`). Drops processed files from `import/` after success.

**`analyze_deck.py`** — Hits the Scryfall API to validate cards, compute color identity, check Commander legality, and flag Game Changers. Writes results to `meta.md` in the deck directory. Has its own `parse_decklist()` (different signature from `import_deck.py`'s version) that only reads the normalized `deck.txt` format — it splits on `\nSIDEBOARD:\n` and blank lines, never raw Moxfield/Deckstats; format detection lives solely in `import_deck.py`.

**Deck storage** — Each deck lives in `decks/<slug>/`:
- `deck.txt` — Moxfield-compatible plain text (mainboard, blank line, then commander)
- `deck.dck` — MTG Forge format; `[Main]` contains non-commander cards, commanders are in a separate `[Commander]` section at the end
- `meta.md` — Generated by `analyze_deck.py`; do not edit manually

**Input format detection** — `is_deckstats()` checks for `Main` / `Sideboard` section headers. Everything else is treated as Moxfield. Commanders in Moxfield format are cards after the final blank line; in Deckstats they are `SB:` lines with `# !Commander` annotation. Forge `.dck` files are detected by extension; slug is derived from `[Metadata] Name=` if present, otherwise from the filename. Collector suffixes (`|SET|[NUM]`) are stripped from card names on import.

**Scryfall API notes** — Cards are fetched in batches of 75 via POST `/cards/collection`; a 0.1 s sleep separates batches. Basic lands are priced at $0; other cards fall back to a cheapest-printing search if no price is returned (each fallback also sleeps 0.1 s). `fetch_game_changer_names()` fetches only the first page of results — not paginated. Double-faced cards have no top-level `mana_cost`; `card_mana_cost()` falls back to `card_faces[0].mana_cost`.

**`meta.md` card list** — Commanders are listed first, then remaining cards sorted by `type_group()` order: Creatures → Planeswalkers → Instants → Sorceries → Artifacts → Enchantments → Lands → Other. Color identity is always displayed in WUBRG order.

**`working/` directory** — Manually maintained notes, not generated. `incomplete-decks.md` tracks decks with fewer than 100 cards; `exclude-list.md` lists cards to avoid adding to any incomplete deck.
