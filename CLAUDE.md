# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Import all decks from import/ folder
python3 import_deck.py

# Import a single deck file
python3 import_deck.py <file.txt>

# Import with an explicit deck name
python3 import_deck.py <file.txt> <deck-name>

# Analyze a deck against Scryfall (by slug or path)
python3 analyze_deck.py <deck-slug>
python3 analyze_deck.py <path/to/deck.txt>
```

No build step, test suite, or external dependencies — pure Python stdlib.

## Architecture

Two scripts, one deck storage convention:

**`import_deck.py`** — Converts plain-text deck exports into the canonical per-deck directory structure. Supports Moxfield and Deckstats input formats (auto-detected). Strips Moxfield timestamps from filenames to derive the slug (e.g. `norin-bounce-20250519-061823.txt` → `norin-bounce`). Drops processed files from `import/` after success.

**`analyze_deck.py`** — Hits the Scryfall API to validate cards, compute color identity, check Commander legality, and flag Game Changers. Writes results to `meta.md` in the deck directory.

**Deck storage** — Each deck lives in `decks/<slug>/`:
- `deck.txt` — Moxfield-compatible plain text (mainboard, blank line, then commander)
- `deck.dck` — MTG Forge format with `[metadata]` / `[Main]` sections
- `meta.md` — Generated by `analyze_deck.py`; do not edit manually

**Input format detection** — `is_deckstats()` checks for `Main` / `Sideboard` section headers. Everything else is treated as Moxfield. Commanders in Moxfield format are cards after the final blank line; in Deckstats they are `SB:` lines with `# !Commander` annotation.
